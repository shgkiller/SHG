MACOS环境下Vue环境搭建以及Element-UI集成

官方网站：https://cn.vuejs.org/

安装清单：

  • Homebrew： Mac系统下的包管理器
  • Node.js: JavaScript运行环境(runtime)
  • npm: Nodejs下的包管理器
  • webpack: Vue的组件是通过 .vue自定义的组件，需要打包成 .js文件
  • Vue-cli: 用来生成模板的Vue工程

打开Mac下的终端

安装Homebrew
/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装成功后查看一下版本信息输入: brew -v
安装Node.js
brew install nodejs

也可以下载后安装,下载地址：https://nodejs.org/en/download/

安装成功后查看一下版本信息输入: node -v
获取Nodejs安装目录访问权限
sudo chmod -R 777 /usr/local/lib/node_modules/
安装npm (淘宝镜像)
因为npm安装插件是从国外服务器下载，受网络影响大，可能出现异常。如果npm的服务器在中国就好了，所以我们乐于分享的淘宝团队干了这事。
原有镜像为官网http://registry.npmjs.org要替换成淘宝镜像
1.npm install -g cnpm --registry=https://registry.npm.taobao.org 安装淘宝镜像
2.npm config set registry https://registry.npm.taobao.org 设置npm默认镜像为淘宝镜像
3.npm config get registry 查看镜像是否改为淘宝镜像
安装webpack
cnpm install webpack -g
全局安装vue脚手架 vue-cli@2
cnpm install vue-cli -g
先创建一个vue项目
执行命令1. cd  路径(想把项目创建在哪个文件夹下面)

vue-cli@2的初始化方式命令如下：
执行命令2. vue init webpack-simple demo1<工程名字不能用中文,不要用大写>

也可执行命令: vue init webpack demo1
这样会询问我们是否安装一些Vue依赖模块，也可以简单(webpack-simple)创建完项目,
需要的其他模块单独安装

当出现如下信息时，vue init已执行完毕
To get started:
   cd firstVueProject(你创建的工程文件名)
   npm install
   npm run dev 
依次执行命令：
执行命令3. cd firstVueProject(你创建的工程文件名)
执行命令4. cnpm install (可一路回车Enter)

单独安装模块: 可安装(或不安装) 
vue 路由模块vue-router
网络请求模块vue-resource
其他模块可继续加在一起执行命令：
cnpm install vue-router vue-resource --save-dev

启动项目
执行命令5. cnpm run dev

至此 vue单独建项 结束
